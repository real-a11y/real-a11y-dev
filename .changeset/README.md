# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works with multi-package repos, or single-package repos to help you version and publish your code. You can find the full documentation for it [in our repository](https://github.com/changesets/changesets).

## How we use changesets in this repo

Every PR that changes one of the publishable `@real-a11y-dev/*` packages should include a changeset. Add one with:

```bash
pnpm changeset
```

You'll be prompted to:

1. Select which packages changed.
2. Choose the bump type per package (`patch`, `minor`, `major`).
3. Write a one-line summary that will land in the package's `CHANGELOG.md`.

The CLI writes the answer to `.changeset/<random-name>.md`. Commit that file alongside your code changes.

## Linked packages

All `@real-a11y-dev/*` packages are **linked** in `config.json` — they bump together so the user-facing version stays consistent across `core`, `ui`, `inspector`, `react`, `testing`, and `storybook-addon`. This matches how the project has been released so far (everything moves on the same beta train).

## Ignored packages

`@real-a11y-dev/semantic-navigator-extension`, `@real-a11y-dev/website`, and the `@real-a11y-dev/example-*` packages are ignored — they are private, never published to npm, and version independently (the extension follows its own Chrome Web Store cadence).

## Releasing

The Changesets GitHub Action will eventually open a "Version Packages" PR that consumes the pending changesets, bumps versions, and writes per-package CHANGELOGs. Merging that PR triggers the `publish.yml` workflow on a pushed tag.

Until that automation is wired up, releases are still cut manually — but new contributions should already include a changeset so we don't have to backfill them later.
